# 数据流与控制流分析

## 概述

本文档完整描述 Face Access System V1.0 的数据流（data flow）和控制流（control flow），覆盖从传感器输入到云端通信的完整链路。所有 IPC 通信通过 `msg_queues.h` 定义的 12 个 FreeRTOS 队列实现，零拷贝传递帧缓冲，各服务独立任务解耦。

### 系统服务

| 模块 | 职责 | 实现状态 |
|------|------|----------|
| `svc_camera` | 摄像头采集（PIR 触发 + 帧分发） | ✅ 已完成 |
| `svc_touch` | 触摸坐标采集（20ms 轮询 FT5406） | ✅ 已完成 |
| `svc_ui` | LVGL 界面渲染 + 摄像头预览 | ✅ 已完成 |
| `svc_face_detect` | 人脸检测（输出边界框） | ⬜ Stub |
| `svc_face_feature` | 人脸特征提取（512 维向量） | ⬜ Stub |
| `svc_face_identify` | 1:N 人脸比对 | ⬜ Stub |
| `svc_perm_manager` | 用户权限管理 + 比对结果处理 | ⬜ Stub |
| `svc_mqtt` | MQTT 通信（日志上报 + 远程指令 + OTA） | ⬜ Stub |
| `svc_log` | 日志管理（环形缓冲 + MQTT 转发） | ⬜ Stub |
| `svc_wdt` | 心跳监控 + 喂狗 | ✅ 已完成 |

### 中间件

| 模块 | 职责 |
|------|------|
| `msg_queues` | IPC 消息队列（12 个通道） |
| `face_engine` | 人脸算法引擎可插拔接口（detect/extract/compare） |
| `db_store` | 人脸特征数据库（NVS 索引 + SD 持久化） |
| `mw_event_bus` | 系统事件总线（FAULT/HEARTBEAT） |
| `log_sink` | 异步日志下沉（环形缓冲） |

## 2 数据流详细分析

### 2.1 camera → ui（预览帧）

| 属性 | 值 |
|------|-----|
| 通道 | `g_q_cam_to_ui` |
| 深度 | 3 |
| 消息类型 | `frame_buffer_t` |
| 数据格式 | RGB888, 800×480×3 = 1.15MB/帧 |
| 传输方式 | **零拷贝**：传递 PSRAM buffer 指针，不复制像素数据 |
| 抽稀比 | `CONFIG_FACE_CAM_FRAME_DIV`（默认 2，每 2 帧取 1 帧） |
| 归还通道 | `g_q_ui_to_cam_frame_ret`（深度:4） |

```
svc_camera task:
  DQBUF 取帧 → fill frame_buffer_t → xQueueSend(g_q_cam_to_ui, 100ms)
  s_in_flight[slot].refs |= 0x01 (bit0=UI)

svc_ui task:
  xQueueReceive(g_q_cam_to_ui, 0) → 更新 lv_image_dsc_t.data
  lv_timer_handler() → render → display_flush_cb → draw_bitmap
  xQueueSend(g_q_ui_to_cam_frame_ret) → 归还上一帧

svc_camera drain:
  xQueueReceive(g_q_ui_to_cam_frame_ret) → refs &= ~0x01
  refs==0 → return_frame → QBUF 归还 V4L2 驱动
```

### 2.2 camera → detect（检测帧）

| 属性 | 值 |
|------|-----|
| 通道 | `g_q_cam_to_det` |
| 深度 | 2 |
| 消息类型 | `frame_buffer_t` |
| 归还通道 | `g_q_det_to_cam_frame_ret`（深度:3） |

camera 同时发送同一帧给 UI 和 detect。`s_in_flight[slot].refs` 用 **bit0=UI、bit1=DET** 引用计数，两个消费者都归还后才 `return_frame`。

### 2.3 touch → ui（触摸数据）

| 属性 | 值 |
|------|-----|
| 通道 | `g_q_touch_to_ui` |
| 深度 | 2 |
| 消息类型 | `touch_msg_t`（嵌入 `dal_touch_point_t` + `timestamp`） |

```
svc_touch task (独立, prio=4, stack=4KB):
  for (;;) {
    touch_ops->read(ctx, pts, 5, &count)  // I2C 读 FT5406 @0x38
    for each pt: fill touch_msg_t { .point=pt, .timestamp }
    xQueueSend(g_q_touch_to_ui, 0ms)       // 非阻塞发送
    count==0 && prev>0 → 发送抬起事件清空状态
    vTaskDelayUntil(20ms)
  }

svc_ui LVGL indev read_cb (~33ms):
  xQueueReceive(g_q_touch_to_ui, &msg, 0)  // 非阻塞读取
  有消息 → data.point = msg.point → LV_INDEV_STATE_PR/RELEASED
  无消息 → LV_INDEV_STATE_RELEASED
```

> `touch_msg_t` 直接嵌入来自 `dal_touch_interface.h` 的 `dal_touch_point_t`（id/x/y/event），消除跨层重复定义。额外添加 `timestamp` 字段作为传输层时间戳。

### 2.4 detect → ui（人脸坐标框）

| 属性 | 值 |
|------|-----|
| 通道 | `g_q_det_to_ui` |
| 深度 | 2 |
| 消息类型 | `detect_to_ui_t` |

```c
typedef struct detect_to_ui_t {
    uint32_t   frame_id;
    uint8_t    face_count;
    face_box_t boxes[FACE_MAX_BOXES];  // face_box_t 复用 face_engine.h 定义（含 float score）
} detect_to_ui_t;
```

svc_ui 收到后在预览画面上叠加人脸框（LVGL rectangle 控件）。

### 2.5 detect → feature（人脸图像预处理）

| 属性 | 值 |
|------|-----|
| 通道 | `g_q_det_to_feature` |
| 深度 | 3 |
| 消息类型 | `detect_to_feature_t` |

```c
typedef struct detect_to_feature_t {
    uint8_t  *buffer;       // PSRAM: 裁剪+对齐人脸图像
    uint32_t  frame_id;     // 关联原始帧
    uint32_t  face_hash;    // 唯一标识（帧ID+序号哈希）
    uint16_t  width;        // 典型 112
    uint16_t  height;       // 典型 112
} detect_to_feature_t;
```

流程：原始帧 ∇ 人脸区域 → 缩放+对齐 → 112×112 RGB → PSRAM → 队列发送。

### 2.6 feature → perm_manager（特征向量）

| 属性 | 值 |
|------|-----|
| 通道 | `g_q_feature_to_perm` |
| 深度 | 3 |
| 消息类型 | `feature_to_perm_t` |

```c
typedef struct feature_to_perm_t {
    float    *features;     // PSRAM: 特征向量
    uint32_t  face_hash;    // 关联 detect_to_feature_t
    uint16_t  dim;          // 维度（512）
} feature_to_perm_t;
```

`svc_face_feature` 完成 CNN 推理后，通过 `face_engine_ops.extract()` 输出 `face_feature_t`（定长 `float vec[512]`），包装为 `feature_to_perm_t` 发送。consumer 处理完须 `free(features)`。

### 2.7 models → PSRAM（CNN 模型加载）

```
Flash 16MB, models SPIFFS 分区 (4MB @0x620000)
  │
  │ esp_partition_mmap(models_partition, &ptr, &handle)
  │ 从 SPIFFS 读 detect.bin + feature.bin → PSRAM
  │
  ▼
PSRAM (heap_caps_malloc SPIRAM | MALLOC_CAP_DMA)
  │
  │ face_engine_init(models_ptr):
  │   detect_model  → esp-dl HumanFaceDetect
  │   feature_model → esp-dl FaceRecognition / MobileFaceNet
  │
  ▼
face_engine_detect() / face_engine_extract() / face_engine_compare()
```

### 2.8 sd → psram（开机加载人脸特征库）

```
SD card (SDMMC / SPI)
  │
  │ db_store_init(): 读取 NVS 索引 → SD 文件遍历 → 反序列化
  │
  ▼
PSRAM (heap_caps_malloc SPIRAM)
  │
  │ feature_db_t {
  │   uint32_t user_ids[256];
  │   float    vectors[256][FACE_FEATURE_DIM];  // 512
  │   uint16_t user_count;
  │ }
```

### 2.9 psram → perm_manager（1:N 比对）

```c
// svc_perm_manager 比对流程：
feature_db_t db;
db_store_get_all(&db);
float best_score = 0;
int   best_id    = -1;

for (int i = 0; i < db.user_count; i++) {
    float score;
    face_engine_compare(&input_feat, &db.vectors[i], &score);
    if (score > best_score) { best_score = score; best_id = db.user_ids[i]; }
}

if (best_score >= CONFIG_FACE_RECOG_THRESH / 1000.0f) {
    // 识别成功 → log_sink_push → relay 开锁 → MQTT 上报
} else {
    // 陌生人 → log_sink_push → MQTT 告警
}
```

### 2.10 perm_manager → log → mqtt（日志链路）

```
svc_perm_manager
  │
  │ log_sink_push(LOG_LVL_INFO, "PERM",
  │     "user %s verified (score=%.2f)", name, score);
  │
  ▼
log_sink 环形缓冲
  │
  │ svc_log 消费任务: 格式化 → ESP_LOGI + (可选) MQTT 转发
  │
  ▼
svc_mqtt
  │ topic: face/{device_id}/event/log
  │ payload: {"type":"auth","user":"张三","score":0.92,"ts":1712345678}
```

### 3.1 pir → camera（人体感应触发）

```
PIR GPIO 双沿中断:
  RISING  (0→1)  →  xTaskNotifyFromISR(s_worker, CAM_NOTIFY_RISING)
  FALLING (1→0)  →  xTaskNotifyFromISR(s_worker, CAM_NOTIFY_FALLING)

camera 三态机:
  SUSPEND ──[RISING]──▶ RUNNING
  RUNNING ──[FALLING]─▶ LEAVE_CONFIRM ──[timeout=8s]──▶ SUSPEND
  LEAVE_CONFIRM ──[RISING]──▶ RUNNING (人又回来了)
  any state  ──[PAUSE cmd]──▶ SUSPEND
```

### 3.2 camera → ui（采集成功开始预览）

```
camera: 第一帧 xQueueSend(g_q_cam_to_ui) 成功 → 无显式通知
ui:     process_camera_frame() 收到帧 → switch_screen(PREVIEW)
        → lv_scr_load(s_scr_preview) + xQueueSend(g_q_ui_to_cam, RESUME)
```

### 3.3 ui → camera（停止采集）

```
ui 触摸进入管理界面:
  xQueueSend(g_q_ui_to_cam, CAM_CMD_PAUSE, 0)
  → camera 收到 → s_state = SUSPEND → 停止 DQBUF
```

typedef enum {

### 3.5 OTA 升级控制流

```
svc_mqtt 收到 topic: face/{device_id}/ota/start
  │ payload: {"version":"2.0","size":1250000,"sha256":"abc...","url":"https://..."}
  │
  ▼
OTA 处理流程:
  1. esp_ota_get_next_update_partition(NULL) → 返回 ota_1 分区
  2. esp_ota_begin(partition, image_size, &handle)
  3. HTTP GET url → 分块 esp_ota_write(handle, data, len)
  4. esp_ota_end(handle) → SHA256 校验
  5. esp_ota_set_boot_partition(partition)
  6. 上报 MQTT: {"status":"ready","next_boot":"ota_1"}
  7. esp_restart()
  8. bootloader 读 otadata → 启动 ota_1 新固件
  9. 新固件 esp_ota_mark_app_valid_cancel_rollback()
```

## 5 Flash 分区与数据存储

### 5.1 分区布局（16MB Flash）

```
0x000000  Bootloader         28KB
0x008000  Partition Table    4KB
0x009000  NVS                24KB   — 系统配置 (WiFi/MQTT/设备ID)
0x00F000  PHY Init           4KB    — RF 校准
0x010000  OTA Data           8KB    — 活跃槽位/启动计数
0x020000  ota_0              3MB    — 固件 A (运行)
0x320000  ota_1              3MB    — 固件 B (OTA 目标)
0x620000  models (SPIFFS)    4MB    — CNN 模型
0xA20000  user_nvs           384KB  — 用户数据 (OTA 后保留)
0xA80000  (空闲)             5.5MB  — 预留
```

### 5.2 数据存储分层

```
热数据 (PSRAM):
  - 当前采集帧 (V4L2 DMA buffer, ~1.15MB × 3)
  - CNN 模型权重 (从 Flash models 分区加载)
  - 特征数据库 (从 SD card 加载, user_count × 512 floats)

温数据 (NVS Flash):
  - 设备配置: device_id, WiFi SSID/密码, MQTT broker URI
  - 用户索引: user_count, user_id 列表

冷数据 (SD Card):
  - 特征向量持久化: user_{id}.bin
  - 抓拍图片 (可选): capture_{ts}.jpg
  - 日志导出 (可选): access_log.csv

远端 (MQTT Server):
  - 开门记录: face/{device_id}/event/unlock
  - 陌生人告警: face/{device_id}/event/stranger
  - OTA 固件分发: face/{device_id}/ota/start
```

## 7 吞吐量与延迟

| 链路 | 速率 | 单次延迟 | 瓶颈 |
|------|------|----------|------|

| OTA 下载 | — | ~10-30s | HTTPS 带宽 |

## 8 生命周期时序

```
开机:
  1. NVS init + 事件循环 + netif
  2. board_v1_init(): 共享 I2C + BSP create + DAL register
  3. msg_queue_init(): 创建 12 个 IPC 队列
  4. 中间件 init: db_store (SD→PSRAM 加载特征库) + face_engine (Flash→PSRAM 加载模型)
  5. Service 注入 + 启动 (init 不触发硬件):
     svc_log.init → svc_wdt.init → svc_camera.init →
     svc_touch.init → svc_ui.init →
     svc_face_detect.init → svc_face_feature.init →
     svc_face_identify.init → svc_perm_manager.init → svc_mqtt.init
  6. Service start (硬件初始化 + 任务创建):
     svc_touch.start → svc_ui.start → svc_camera.start
  7. 喂狗尾循环 (app_main while(1) { vTaskDelay + esp_task_wdt_reset })

运行:
  空闲: camera=SUSPEND, ui=IDLE, 人脸 pipeline 空转等待队列消息
  有人靠近: PIR↑ → camera=RUNNING → 采集帧 → 分发 UI+detect
  有人离开: PIR↓ → LEAVE_CONFIRM(8s timeout) → SUSPEND

关机/重启:
  各 service.stop: 反注册 ISR → 归还帧 → deinit 硬件 → 删除任务
  msg_queue_deinit(): 删除 12 个队列
```

# 自定义人脸特征提取模型部署指南

## 概述

本文档描述如何将自行训练的 PyTorch/TensorFlow 人脸特征提取模型（MobileFaceNet / ResNet / EfficientNet 等）转换为 esp-dl FlatBuffers 格式（`.espdl`），并集成到 Face Access System 固件中。

### 系统架构回顾

```
人脸识别 pipeline（不需要独立的"比对模型"）:

camera → svc_face_detect (detect)
           │  face_box_t[]
           ▼
         crop + align → 112×112 RGB
           │
           ▼
         svc_face_feature (extract)       ← 你的自定义模型在这里
           │  CNN 推理 → float[512]
           ▼
         svc_perm_manager                 ← 纯数学, 不需要模型
           │  cosine_similarity(feat, db.vectors[i])
           │  遍历 PSRAM 中 1000 个特征向量
           │  best_score >= THRESH → 识别成功
```

## 1 模型格式要求

### 1.1 输入

| 参数 | 值 |
|------|-----|
| 图像尺寸 | 112×112 (或 96×96 / 128×128) |
| 通道 | 3 (RGB) |
| 像素格式 | RGB888 或 RGB565 |
| 归一化 | 根据训练时配置, esp-dl 在 ImagePreprocessor 中处理 |

### 1.2 输出

| 参数 | 值 |
|------|-----|
| 特征维度 | 512 (推荐, 与 `FACE_FEATURE_DIM` 一致) |
| 类型 | float32 |
| 后处理 | L2 归一化 (可在 CPU 侧完成) |

### 1.3 支持的算子

esp-dl 支持的算子集合（转为 FlatBuffers 时需确保所有算子有对应实现）：

```
Conv2D, DepthwiseConv2D, Conv2DTranspose
MaxPool2D, AvgPool2D, GlobalAvgPool2D
ReLU, ReLU6, LeakyReLU, PReLU, Sigmoid, Tanh, HardSwish
Add, Mul, Sub, Div, Concat
BatchNorm (fused to Conv), Dropout (removed at inference)
FullyConnected, Softmax
Resize (nearest/bilinear), Pad, Transpose, Reshape
```

> 不被支持的算子（如 LayerNorm、GELU、Attention）需要在导出 ONNX 时**替换或分解**。

## 2 模型转换流程

### Step 1: PyTorch/TensorFlow → ONNX

```python
# PyTorch 示例: MobileFaceNet → ONNX
import torch

model = MobileFaceNet(num_features=512)
model.load_state_dict(torch.load("mfn_checkpoint.pth"))
model.eval()

dummy = torch.randn(1, 3, 112, 112)  # NCHW

torch.onnx.export(
    model, dummy, "mfn.onnx",
    input_names=["input"],
    output_names=["feature"],
    opset_version=11,           # esp-dl 支持 opset 11
    dynamic_axes=None,          # 固定尺寸, 不要动态轴
    do_constant_folding=True,
)
```

```python
# 验证 ONNX
import onnx
model = onnx.load("mfn.onnx")
onnx.checker.check_model(model)
print("ONNX OK")
```

### Step 2: ONNX → esp-dl FlatBuffers (.espdl)

使用 **esp-ppq** 进行 S8 量化并直接导出 ESP-DL 标准格式：

```bash
# 安装 esp-ppq
pip install git+https://github.com/espressif/esp-ppq.git

# 运行转换 (脚本在 tools/convert_feature_model.py)
source ~/esp/esp-idf/export.sh
python3 tools/convert_feature_model.py
```

或者手动调用 esp-ppq API：

```python
from ppq import espdl_quantize_onnx, TargetPlatform
import onnx, numpy as np

model = onnx.load("feature_model.onnx")
calib = np.load("calib_images.npy")

espdl_quantize_onnx(
    onnx_model=model,
    calib_dataset=calib,
    target=TargetPlatform.ESP32P4,   # 目标芯片
    output_path="mfn_s8_v1.espdl",
)
```

### 3.4 face_feature_model.hpp

```cpp
#pragma once
#include "dl_model_base.hpp"

class FaceFeatureModel {
public:
    FaceFeatureModel();
    ~FaceFeatureModel();

    /**
     * @brief 运行特征提取
     * @param img      输入图像 (RGB888, 112×112)
     * @param feature  输出特征向量 (float[512], 调用方分配)
     * @return 0 成功
     */
    int run(const dl::image::img_t &img, float *feature);

private:
    dl::Model *m_model;
    dl::image::ImagePreprocessor *m_preprocessor;
};
```

### 3.5 face_feature_model.cpp

```cpp
#include "face_feature_model.hpp"

#ifdef CONFIG_FACE_FEATURE_MODEL_IN_FLASH_RODATA
extern const uint8_t mfn_espdl[] asm("_binary_mfn_s8_v1_espdl_start");
static const char *path = (const char *)mfn_espdl;
#else
static const char *path = "human_face_feat";
#endif

FaceFeatureModel::FaceFeatureModel()
{
    m_model = new dl::Model(
        path, "mfn_s8_v1.espdl",
        static_cast<fbs::model_location_type_t>(CONFIG_FACE_FEATURE_MODEL_LOCATION));

#if CONFIG_IDF_TARGET_ESP32P4
    m_preprocessor = new dl::image::ImagePreprocessor(
        m_model, {127.5f, 127.5f, 127.5f}, {0.0078125f, 0.0078125f, 0.0078125f},
        dl::image::DL_IMAGE_CAP_RGB_SWAP);
#endif
}

FaceFeatureModel::~FaceFeatureModel()
{
    delete m_preprocessor;
    delete m_model;
}

int FaceFeatureModel::run(const dl::image::img_t &img, float *feature)
{
    m_preprocessor->preprocess(img);
    m_model->run();

    // 从模型输出 tensor 拷贝特征向量
    dl::TensorBase *output = m_model->get_output(0);
    float *data = reinterpret_cast<float *>(output->get_element_ptr());
    int dim = output->get_size();

    // L2 归一化
    float norm = 0.0f;
    for (int i = 0; i < dim; i++) norm += data[i] * data[i];
    norm = 1.0f / sqrtf(norm);
    for (int i = 0; i < dim; i++) feature[i] = data[i] * norm;

    return dim;
}
```

## 4 集成到 face_engine

### 4.1 更新 esp_dl_adapter.cpp

```cpp
// 在现有 esp_dl_adapter.cpp 中添加:
#include "face_feature_model.hpp"

static FaceFeatureModel *s_feat_model = NULL;

static dal_err_t dl_extract(void *ctx, const face_image_t *crop,
                             face_feature_t *out_feat)
{
    (void)ctx;
    if (s_feat_model == NULL || crop == NULL || out_feat == NULL) {
        return DAL_ERR_INVALID;
    }

    dl::image::img_t dl_img = {
        .data = (void *)crop->data,
        .width = crop->width,
        .height = crop->height,
        .pix_type = dl::image::DL_IMAGE_PIX_TYPE_RGB888,
    };

    int dim = s_feat_model->run(dl_img, out_feat->vec);
    out_feat->dim = (uint16_t)dim;
    return (dim > 0) ? DAL_OK : DAL_ERR_HW;
}

// cosine similarity — 纯数学, 不需要模型
static dal_err_t dl_compare(void *ctx, const face_feature_t *a,
                             const face_feature_t *b, float *out_score)
{
    (void)ctx;
    if (a == NULL || b == NULL || out_score == NULL || a->dim != b->dim) {
        return DAL_ERR_INVALID;
    }

    float dot = 0.0f;
    for (uint16_t i = 0; i < a->dim; i++) {
        dot += a->vec[i] * b->vec[i];
    }
    // 特征已 L2 归一化, dot ∈ [-1, 1], 映射到 [0, 1]
    *out_score = (dot + 1.0f) * 0.5f;
    return DAL_OK;
}

// 更新 ops 表:
static const face_engine_ops_t s_dl_ops = {
    .init    = dl_init,
    .deinit  = dl_deinit,
    .detect  = dl_detect,
    .extract = dl_extract,    // ← 新增
    .compare = dl_compare,    // ← 新增 (纯数学)
};
```

### 4.2 dl_init 中添加特征模型初始化

```cpp
static dal_err_t dl_init(void *ctx)
{
    (void)ctx;
    // ... 现有 HumanFaceDetect 初始化 ...

    s_feat_model = new FaceFeatureModel();
    if (s_feat_model == NULL) {

{
    (void)ctx;
    if (s_model)     { delete s_model;     s_model     = NULL; }
    if (s_feat_model){ delete s_feat_model;s_feat_model = NULL; }
    return DAL_OK;
}
```

## 5 余弦相似度（不需要模型）

```c
/**
 * @brief 余弦相似度 — 纯 CPU 运算
 *
 * @note 输入向量必须预先 L2 归一化 (|a|=|b|=1)。
 *       输出 score ∈ [0, 1], 1=完全相同, 0=完全相反。
 *
 * 复杂度: O(dim), dim=512 时约 512 次乘加 + 1 次除法
 * 耗时: <1µs (ESP32-P4 @400MHz)
 */
static inline float cosine_similarity(const float *a, const float *b, int dim)
{
    float dot = 0.0f;
    for (int i = 0; i < dim; i++) {
        dot += a[i] * b[i];
    }
    return (dot + 1.0f) * 0.5f;   // [-1,1] → [0,1]
}
```

## 6 模型文件大小估算

| 格式 | 大小 | 说明 |
|------|------|------|
| PyTorch (.pth) | ~4-8MB | float32 权重 |
| ONNX | ~4-8MB | float32 |
| esp-dl S8 (.espdl) | ~1-2MB | int8 量化 + FlatBuffers 打包 |
| esp-dl S16 (.espdl) | ~2-4MB | int16 量化 |

对于 512 维 MobileFaceNet (参数量 ~2M):
- float32: ~8MB
- int8 S8: **~1.5MB** ← 推荐 (精度损失 <0.5%)

## 8 不需要"人脸比对模型"的原因

特征提取后，`svc_perm_manager` 直接用余弦相似度比对两个 512 维向量：

```
新采集人脸 → extract → feat_A[512]
                        │
PSRAM 特征库 ──────────┘
  db.vectors[0][512] → cos(feat_A, db[0]) = 0.83
  db.vectors[1][512] → cos(feat_A, db[1]) = 0.21
  ...
  db.vectors[999][512] → cos(feat_A, db[999]) = 0.95 ★ best

best_score=0.95 >= THRESH(0.6) → user_999 识别成功 → 开锁
```

这是 **metric learning** 的标准流程。特征提取 CNN 已经将人脸映射到了一个**余弦空间**——同一人的两张照片在这个空间中距离近（cos→1），不同人距离远（cos→0）。比对阶段只需要做余弦相似度计算，不需要额外的神经网络。

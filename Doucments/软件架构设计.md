# 人脸识别系统 — 软件架构设计

## 1. 架构概述

本规范定义了一套基于"设备管理 + 接口注入"的嵌入式分层架构。通过严格的分层设计与契约化编程，实现硬件驱动与业务逻辑的彻底解耦，确保系统的可测试性、可扩展性与长期可维护性。

### 1.1 架构图

```
+----------------------------+
|     Service                |   业务逻辑层（依赖注入，由 Assembler 组装）
+----------------------------+
|     Middleware              |   中间件层（人脸算法、协议栈、通用组件）
+----------------------------+
|       DAL                  |   设备抽象层（纯业务语义契约 + 设备注册表）
+----------------------------+
|       BSP                  |   板级适配层（硬件适配 + ops/ctx 绑定，不负责注册）
+----------------------------+
|   FreeRTOS / ESP-IDF       |   基础设施（RTOS + 芯片 SDK，直接调用）
+----------------------------+
```

### 1.2 核心思想

- **设备管理**：BSP 设备驱动仅负责创建 ops + ctx 绑定（`create()` 返回 `ops_t*`），板级组装器（`bsp_v1_init`）统一完成 DAL 注册。DAL 仅维护"业务功能"注册表，不感知具体芯片型号。
- **接口注入**：Service 声明能力需求（Deps），Assembler 从 DAL 获取实现并完成依赖注入。
- **延迟初始化**：`create()` 不触发硬件初始化，仅绑定 struct。硬件初始化由 Service 层通过 `ops->init()` 按需触发，降低启动功耗与启动失败风险。
- **参数隔离**：所有跨层接口参数必须为纯业务语义，严禁泄露底层硬件类型（`gpio_num_t`、`i2c_port_t` 等）。
- **错误码隔离**：底层 HAL/SDK 错误码（如 `esp_err_t`）必须在 BSP 边界翻译为统一的 `dal_err_t`，禁止透传到 DAL/SVC。
- **无冗余抽象**：不引入 PAL/OSAL 中间层，BSP 直接调用 FreeRTOS 原生 API 和 ESP-IDF 驱动库。

### 1.3 各层职责

| 层 | 职责 | 是否感知硬件 |
|----|------|--------------|
| **Service** | 业务逻辑层。通过依赖注入获得 DAL ops，编排业务流程。对底层芯片型号完全盲化。 | 否 |
| **Middleware** | 中间件层。与具体硬件无关的通用组件，如人脸检测/识别算法（ESP-DL）、数据格式转换、滤波算法。 | 否 |
| **DAL** | 设备抽象层。定义纯业务语义接口契约（`dal_xxx_interface.h`）+ 设备注册表管理（`dal_xxx.h/.c`）。不依赖任何 BSP。 | 否 |
| **BSP** | 板级适配层。针对具体板卡，直接调用 ESP-IDF / FreeRTOS API 驱动真实芯片，实现 DAL ops，将 ctx 注入 ops 并返回。**不负责 DAL 注册与硬件初始化**——注册由板级组装器完成，初始化由 Service 层按需触发。 | 是 |

> **说明**：本项目不引入 PAL（平台屏蔽层）与 OSAL（OS 屏蔽层）。BSP 直接 `#include "driver/gpio.h"`、`"freertos/FreeRTOS.h"` 等 ESP-IDF 头文件，减少无意义的间接调用层。

## 3. 核心架构实现

### 3.1 DAL 设备管理实现

#### 3.1.1 公共错误码定义

> **关键约束**：`dal_err_t` 是 DAL/SVC 层唯一允许的错误码。BSP 必须在 ops 边界把底层码（`esp_err_t`）翻译为 `dal_err_t`，原始码不得透传。

```c
/**
 * @file    dal_err.h
 * @brief   DAL 公共错误码定义
 *
 * @author  xiamu
 * @version 1.0
 */
#ifndef DAL_ERR_H
#define DAL_ERR_H

#ifdef __cplusplus
extern "C" {
#endif

/** DAL 统一错误码 */
typedef enum {
    DAL_OK              =  0,  /**< 成功 */
    DAL_ERR_INVALID     = -1,  /**< 参数非法 */
    DAL_ERR_NO_MEM      = -2,  /**< 无内存 / 注册表满 */
    DAL_ERR_NOT_FOUND   = -3,  /**< 设备未注册 */
    DAL_ERR_BUSY        = -4,  /**< 设备忙 */
    DAL_ERR_TIMEOUT     = -5,  /**< 超时 */
    DAL_ERR_HW          = -6,  /**< 底层硬件错误（已翻译，不泄露原始码） */
    DAL_ERR_STATE       = -7,  /**< 状态非法（如重复注册） */
    DAL_ERR_UNSUPPORTED = -8,  /**< 操作不支持 */
} dal_err_t;

#ifdef __cplusplus
}
#endif
#endif /* DAL_ERR_H */
```

#### 3.1.4 设备管理实现

```c
/**
 * @file    dal_display.c
 * @brief   显示设备 DAL 管理实现（注册表 + FreeRTOS 互斥保护）
 *
 * @author  xiamu
 * @version 1.0
 */
#include "dal_display.h"
#include "freertos/FreeRTOS.h"
#include "freertos/semphr.h"
#include <string.h>

#define MAX_DEVICES 4

typedef struct {
    const char              *name;
    const dal_display_ops_t *ops;
    void                    *ctx;
} device_entry_t;

static device_entry_t    s_registry[MAX_DEVICES];
static size_t            s_count = 0;
static SemaphoreHandle_t s_mutex = NULL;

/* 注册表互斥锁在模块加载期一次性创建 */
__attribute__((constructor))
static void registry_once_init(void)
{
    s_mutex = xSemaphoreCreateMutex();
}

dal_err_t dal_display_register(const char *name,
                               const dal_display_ops_t *ops,
                               void *ctx)
{
    if (!name || !ops) {
        return DAL_ERR_INVALID;
    }

    xSemaphoreTake(s_mutex, portMAX_DELAY);
    if (s_count >= MAX_DEVICES) {
        xSemaphoreGive(s_mutex);
        return DAL_ERR_NO_MEM;
    }
    for (size_t i = 0; i < s_count; i++) {
        if (strcmp(s_registry[i].name, name) == 0) {
            xSemaphoreGive(s_mutex);
            return DAL_ERR_STATE;
        }
    }

    s_registry[s_count].name = name;
    s_registry[s_count].ops  = ops;
    s_registry[s_count].ctx  = ctx;
    s_count++;
    xSemaphoreGive(s_mutex);
    return DAL_OK;
}

dal_err_t dal_display_get(const char *name,
                          const dal_display_ops_t **ops,
                          void **ctx)
{
    if (!name || !ops || !ctx) {
        return DAL_ERR_INVALID;
    }

    *ops = NULL;
    *ctx = NULL;

    xSemaphoreTake(s_mutex, portMAX_DELAY);
    for (size_t i = 0; i < s_count; i++) {
        if (strcmp(s_registry[i].name, name) == 0) {
            *ops = s_registry[i].ops;
            *ctx = s_registry[i].ctx;
            xSemaphoreGive(s_mutex);
            return DAL_OK;
        }
    }
    xSemaphoreGive(s_mutex);
    return DAL_ERR_NOT_FOUND;
}
```

> **设计要点**：
> - 互斥锁在模块初始化期一次性创建，使用 FreeRTOS 原生 `xSemaphoreCreateMutex()`，杜绝懒初始化的竞态。
> - `dal_display_get` 合并 ops 与 ctx 一次取出，避免分两次调用期间设备被注销导致的不一致。

### 3.2 BSP 单驱动 — 以继电器为例

> **规范范式**：BSP 通过 `bsp_xxx_create()` 创建静态 ops + ctx 绑定并返回 `ops_t*`。硬件初始化全部封装在 `ops->init()` 中，由 Service 层按需调用。DAL 注册由板级组装器（`bsp_v1_init`）完成。

```c
/**
 * @file    bsp_relay.c
 * @brief   门锁继电器 BSP — ops + ctx 绑定
 *
 * @note    仅负责创建 ops + ctx 绑定，不调用 DAL 注册 API。
 *          注册由板级组装器完成，硬件初始化由 Service 层按需触发。
 *          引脚号仅在此 .c 内可见，Service 层完全盲化。
 *
 * @author  xiamu
 * @version 1.1
 */
#include "bsp_relay.h"
#include "dal_relay_interface.h"
#include "driver/gpio.h"
#include <string.h>

/* ================================================================
 *  硬件上下文（BSP 私有，仅此处知道引脚号）
 * ================================================================ */
typedef struct {
    gpio_num_t pin;
    uint32_t   active_level;
} bsp_relay_ctx_t;

/* ================================================================
 *  esp_err_t → dal_err_t 翻译（BSP 边界）
 * ================================================================ */
static dal_err_t relay_err(esp_err_t e) {
    return (e == ESP_OK) ? DAL_OK : DAL_ERR_HW;
}

/* ================================================================
 *  ops 实现（BSP 私有）
 * ================================================================ */
static dal_err_t relay_set(void *ctx, bool on)
{
    bsp_relay_ctx_t *s = (bsp_relay_ctx_t *)ctx;
    return relay_err(gpio_set_level(s->pin,
                     on ? s->active_level : 1 - s->active_level));
}

static dal_err_t relay_get(void *ctx, bool *out)
{
    bsp_relay_ctx_t *s = (bsp_relay_ctx_t *)ctx;
    if (!out) return DAL_ERR_INVALID;
    *out = (gpio_get_level(s->pin) == (int)s->active_level);
    return DAL_OK;
}

static dal_err_t relay_init(void *ctx)
{
    bsp_relay_ctx_t *s = (bsp_relay_ctx_t *)ctx;

    gpio_config_t cfg = {
        .pin_bit_mask = BIT64(s->pin),
        .mode         = GPIO_MODE_OUTPUT,
        .pull_up_en   = GPIO_PULLUP_DISABLE,
        .pull_down_en = GPIO_PULLDOWN_DISABLE,
        .intr_type    = GPIO_INTR_DISABLE,
    };
    if (gpio_config(&cfg) != ESP_OK) return DAL_ERR_HW;

    /* 上电释放 */
    gpio_set_level(s->pin, 1 - s->active_level);
    return DAL_OK;
}

/* ================================================================
 *  静态 ctx + ops（create() 注入 ctx）
 * ================================================================ */
static bsp_relay_ctx_t s_ctx;
static dal_relay_ops_t s_ops;

dal_relay_ops_t* bsp_relay_create(void)
{
    memset(&s_ctx, 0, sizeof(s_ctx));
    s_ctx.pin          = GPIO_NUM_2;
    s_ctx.active_level = 1;

    s_ops.init = relay_init;
    s_ops.set  = relay_set;
    s_ops.get  = relay_get;
    s_ops.ctx  = &s_ctx;          /* ctx 注入 ops */

    return &s_ops;
}
```

```c
/**
 * @file    bsp_relay.h
 * @brief   门锁继电器 BSP 公共接口
 *
 * @note    仅负责创建 ops + ctx 绑定，不调用 DAL 注册 API。
 *          注册动作由板级组装器完成。
 *
 * @author  xiamu
 * @version 1.1
 */
#ifndef BSP_RELAY_H
#define BSP_RELAY_H

#include "dal_relay_interface.h"

#ifdef __cplusplus
extern "C" {
#endif

/**
 * @brief 创建继电器 ops + ctx 绑定
 *
 * 使用静态分配的 ctx 与 ops，将 ctx 注入 ops->ctx。
 * 返回指针由调用方通过 dal_relay_register() 注册到 DAL，
 * 再由 ops->init() 完成硬件初始化。
 *
 * @return 非 NULL 成功，NULL 表示内部错误
 */
dal_relay_ops_t* bsp_relay_create(void);

#ifdef __cplusplus
}
#endif
#endif /* BSP_RELAY_H */
```

> **设计要点**：
> - `create()` 仅做 struct 绑定，零副作用。ctx 与 ops 均为静态分配，无 malloc 开销。
> - 硬件初始化（GPIO 配置、默认电平）封装在 `ops->init()`，由 Service 层按需调用。
> - BSP 设备驱动**不包含** `dal_relay.h`，**不调用** `dal_relay_register()`。注册由板级组装器统一执行。
> - `ops->ctx` 将私有上下文注入接口，调用方可通过 `ops->init(ops->ctx)` 完成初始化。

### 3.3 BSP 显示驱动 — ILI9488 SPI 显示屏

> **适用场景**：较复杂的外设驱动，涉及 SPI DMA + PWM 背光。与继电器模式一致，采用 `create() + ops->init()` 模式。

**核心要点**（完整代码见 `components/bsp/bsp_display_2.4/`）：

- **静态 ctx + ops**：`static bsp_display_ctx_t s_display_ctx` + `static dal_display_ops_t s_display_ops`
- **`bsp_display_2_4_create()`**：memset ctx、绑定 `disp_init/disp_flush/disp_bl` 函数指针、注入 `ops.ctx = &s_display_ctx`、返回 `&s_display_ops`
- **`disp_init()`**：包含完整硬件初始化——背光 PWM 配置、ILI9488 SPI 初始化、DMA 安全行缓冲分配（解决 SPI DMA 无法直接访问 PSRAM 的问题）、分块 flush
- **BSP 不包含 `dal_display.h`**，不调用 `dal_display_register()`

```c
/* 背光 — LEDC PWM */
static void bl_init(void)
{
    ledc_timer_config_t t = {
        .speed_mode = LEDC_LOW_SPEED_MODE, .duty_resolution = LEDC_TIMER_10_BIT,
        .timer_num = LEDC_TIMER_0, .freq_hz = 5000, .clk_cfg = LEDC_AUTO_CLK,
    };
    ledc_timer_config(&t);
    ledc_channel_config_t ch = {
        .gpio_num = BSP_LCD_LEDC_PIN, .speed_mode = LEDC_LOW_SPEED_MODE,
        .channel = LEDC_CHANNEL_0, .timer_sel = LEDC_TIMER_0, .duty = 0, .hpoint = 0,
    };
    ledc_channel_config(&ch);
}

/* DAL ops — init 包含全部硬件初始化 */
static dal_err_t disp_init(void *ctx, const dal_display_config_t *cfg)
{
    bsp_display_ctx_t *s = (bsp_display_ctx_t *)ctx;
    bl_init();   /* 背光 PWM 一次性配置 */

    /* ILI9488 SPI 初始化 + DMA 行缓冲分配 + 首次 bl_set */
    // ...（详见源码）
    return DAL_OK;
}

static dal_err_t disp_flush(void *ctx, const uint8_t *buf, size_t len) { /* 分块 DMA 传输 */ }
static dal_err_t disp_bl(void *ctx, uint8_t pct) { bl_set(pct); return DAL_OK; }

/* create — 纯 struct 绑定 */
static bsp_display_ctx_t s_display_ctx;
static dal_display_ops_t s_display_ops;

dal_display_ops_t* bsp_display_2_4_create(void)
{
    memset(&s_display_ctx, 0, sizeof(s_display_ctx));
    s_display_ops.init          = disp_init;
    s_display_ops.flush         = disp_flush;
    s_display_ops.set_backlight = disp_bl;
    s_display_ops.ctx           = &s_display_ctx;
    return &s_display_ops;
}
```

```c
/**
 * @file    svc_ui.c
 * @brief   UI Service 业务逻辑实现

#include <string.h>

static svc_ui_deps_t     s_deps;

static bool              s_inited = false;

dal_err_t svc_ui_init(const svc_ui_deps_t *deps)
{
    if (!deps || !deps->display_ops || !deps->display_ops->flush_frame) {
        return DAL_ERR_INVALID;
    }

    memcpy(&s_deps, deps, sizeof(s_deps));

    s_mutex = xSemaphoreCreateMutex();
    if (!s_mutex) return DAL_ERR_NO_MEM;

    s_inited = true;
    return DAL_OK;
}
```

### 3.5 Assembler 组装实现

> **职责**：板级组装器（`bsp_v1_init`）负责**按硬件依赖顺序**依次调用 `bsp_xxx_create()` + `dal_xxx_register()`。不执行任何硬件初始化——那由 Service 层通过 `ops->init()` 按需触发。

```c
/**
 * @file    bsp_v1.c
 * @brief   Board V1 板级组装器 — create + register 统一入口
 *
 * @note    按硬件依赖顺序依次调用各 BSP create() 并 dal_xxx_register()。
 *          硬件初始化由 Service 层通过 ops->init() 按需触发。
 *
 * @author  xiamu
 * @version 1.1
 */
#include "bsp_v1.h"
#include "bsp_config.h"
#include "bsp_relay.h"
#include "bsp_pir_hcsr501.h"
#include "bsp_display_2.4.h"
#include "bsp_touch_xpt2046.h"
#include "bsp_cam_ov5640.h"
#include "bsp_microsd.h"
#include "dal_display.h"
#include "dal_camera.h"
#include "dal_pir.h"
#include "dal_relay.h"
#include "dal_touch.h"
#include "dal_sd.h"
#include "driver/spi_master.h"
#include "esp_log.h"

static const char *TAG = "bsp_v1";

dal_err_t bsp_v1_init(void)
{
    dal_err_t ret;

    /* 1. 继电器 — create + register，硬件 init 由 service 按需触发 */
    {
        dal_relay_ops_t *relay = bsp_relay_create();
        ret = dal_relay_register("door_lock", relay, relay->ctx);
        if (ret != DAL_OK) ESP_LOGW(TAG, "relay reg: %d", ret);
    }

    /* 2. PIR — create + register */
    {
        dal_pir_ops_t *pir = bsp_pir_hcsr501_create();
        ret = dal_pir_register("main_pir", pir, pir->ctx);
        if (ret != DAL_OK) ESP_LOGW(TAG, "pir reg: %d", ret);
    }

    /* 3. microSD — create + register（SPI 总线尚未配置默认引脚） */
    {
        dal_sd_ops_t *sd = bsp_microsd_create();
        ret = dal_sd_register("sdcard", sd, sd->ctx);
        if (ret != DAL_OK) ESP_LOGW(TAG, "sdcard reg: %d", ret);
    }

    /* 4. SPI2 总线 — 共享总线初始化（display + touch 共用） */
    spi_bus_config_t bus = {
        .sclk_io_num     = BSP_SPI_SCLK_PIN,
        .mosi_io_num     = BSP_SPI_MOSI_PIN,
        .miso_io_num     = BSP_SPI_MISO_PIN,
        .max_transfer_sz = BSP_SPI_TRAN_MAX_SIZE,
    };
    ret = spi_bus_initialize(BSP_SPI_HOST, &bus, BSP_SPI_DMA_CHAN);
    if (ret != ESP_OK) { ESP_LOGE(TAG, "SPI bus fail"); return DAL_ERR_HW; }

    /* 5. 显示屏 — create + register */
    {
        dal_display_ops_t *display = bsp_display_2_4_create();
        ret = dal_display_register("main_lcd", display, display->ctx);
        if (ret != DAL_OK) { ESP_LOGE(TAG, "display reg: %d", ret); return ret; }
    }

    /* 6. 触摸 — create + register */
    {
        dal_touch_ops_t *touch = bsp_touch_xpt2046_create();
        ret = dal_touch_register("main_touch", touch, touch->ctx);
        if (ret != DAL_OK) ESP_LOGW(TAG, "touch reg: %d", ret);
    }

    /* 7. 摄像头 — create + register */
    {
        dal_camera_ops_t *cam = bsp_cam_ov5640_create();
        ret = dal_camera_register("main_cam", cam, cam->ctx);
        if (ret != DAL_OK) ESP_LOGW(TAG, "camera reg: %d", ret);
    }

    ESP_LOGI(TAG, "BSP init done");
    return DAL_OK;
}
```

## 4. 架构规范约束

### 4.1 依赖方向约束

**编译期依赖（头文件包含）**：

```
main.c     → BSP(接口头) + DAL(管理头) + Service(契约头)
bsp_v1.c   → BSP(接口头) + DAL(管理头) + bsp_config.h
Service    → DAL(接口契约头) + Middleware(接口头)
BSP        → DAL(接口契约头) + ESP-IDF(驱动头) + FreeRTOS
DAL        → FreeRTOS + dal_err.h        （DAL 严禁依赖任何 BSP）
Middleware → DAL(接口契约头)              （Middleware 严禁依赖 BSP）
```

**关键变化**：BSP 设备驱动不再包含 DAL 管理头（`dal_xxx.h`），仅包含接口契约头（`dal_xxx_interface.h`）。注册动作由板级组装器通过管理头完成。

**注册期 / 运行期调用**：

```
bsp_v1_init → bsp_xxx_create()     （BSP 返回 ops + ctx 绑定）
bsp_v1_init → dal_xxx_register()   （板级组装器完成 DAL 注册）
main        → dal_xxx_get()        （取 ops+ctx 注入 Service）
main        → svc_xxx_init(&deps)  （注入依赖）
Service     → ops->init(ops->ctx)  （按需触发硬件初始化）
```

**禁止的依赖**：

- Service → DAL 管理头（`dal_xxx.h`）—— 只允许 `dal_xxx_interface.h`
- Service → BSP / ESP-IDF / 硬件 SDK 头文件
- BSP 设备驱动 → DAL 管理头（`dal_xxx.h`）—— BSP 不负责注册
- DAL → BSP —— DAL 不得反向依赖任何具体 BSP，否则失去设备无关性
- Middleware → BSP —— 中间件不得直接访问硬件

### 4.2 头文件包含规范

#### 4.2.1 Service 层禁止包含

```c
// ❌ 错误示例
#include "dal_display.h"       // 禁止依赖 DAL 管理头
#include "bsp_xxx.h"           // 禁止依赖 BSP
#include "driver/gpio.h"       // 禁止依赖 ESP-IDF 驱动头
#include "freertos/task.h"     // 禁止直接使用 FreeRTOS（API 泄露）
```

#### 4.2.2 BSP 设备驱动禁止包含

```c
// ❌ 错误示例 — BSP 设备驱动不得包含 DAL 管理头
#include "dal_relay.h"         // 禁止！BSP 不负责注册
#include "dal_display.h"       // 禁止！注册由板级组装器完成
```

BSP 仅包含 `dal_xxx_interface.h`（获取 ops 类型定义），不包含 `dal_xxx.h`（管理 API）。

#### 4.2.3 Service 层允许包含

```c
// ✅ 正确示例
#include "svc_xxx.h"                // 自身依赖契约
#include "dal_xxx_interface.h"      // 仅允许依赖 DAL 接口契约
#include "mw_xxx.h"                 // 中间件接口
```

### 4.4 生命周期与初始化顺序

系统启动必须遵循以下严格顺序：

```
1. FreeRTOS 内核启动（调度器就绪）
2. BSP 共用资源初始化（SPI 总线等，在 bsp_v1_init 中按依赖顺序初始化）
3. bsp_v1_init：依次 bsp_xxx_create() + dal_xxx_register() → 注册表填充完成
4. main：dal_xxx_get() 取出 ops+ctx → 注入各 Service init(&deps)
5. Service 按需调用 ops->init(ctx, &cfg) → 触发实际硬件初始化
6. 启动业务任务 / 主循环
```

- **注册表必须在任何 Service init 之前完成填充**，否则 `dal_get` 返回 `DAL_ERR_NOT_FOUND`。
- **硬件初始化延迟到 Service 层按需触发**：`create()` 不驱动硬件，`register()` 不驱动硬件。只有 Service 调用 `ops->init()` 时才真正操作 GPIO/SPI/I2C。
- 共享总线（如 SPI）在 BSP create 之前由板级组装器初始化，create 仅绑定配置参数。
- 析构顺序：先停业务任务 → Service deinit（可调用 `ops->deinit()`） → 注销 DAL 条目。

## 5. 接口设计防泄漏检查清单

为防止硬件细节通过参数或返回值泄露到上层，所有 DAL/SVC 接口设计必须通过以下检查：

- [ ] 接口参数中无硬件枚举（如 `i2c_port_t`、`gpio_num_t`、`spi_host_device_t`）
- [ ] 接口参数中无硬件寄存器结构体指针（如 `spi_transaction_t *`）
- [ ] 复杂硬件配置已替换为纯业务语义结构体或不透明指针 `void *ctx`
- [ ] 返回值统一为 `dal_err_t`，无底层 `esp_err_t` 等透传
- [ ] BSP 在 ops 边界完成错误码翻译（`esp_err_t` → `dal_err_t`）
- [ ] Service 层编译不依赖任何 BSP / ESP-IDF 驱动头文件
- [ ] DAL 接口契约头不 `#include` 任何硬件 SDK 头文件
- [ ] Middleware 层编译不依赖 BSP / 硬件驱动头

## 6. 并发与中断约束

- **DAL ops 默认运行在任务上下文**，禁止在 ISR 中直接调用 ops（多数 ops 含阻塞/互斥）。
  如需 ISR 触发，应在 ISR 中仅投递消息到 FreeRTOS 队列，由任务上下文消费并调用 ops。
  例：PIR 传感器 ISR → `xQueueSendFromISR()` → PIR Task → `dal_pir_get_state()`
- **DAL 注册表内部用 FreeRTOS 互斥锁保护**，`register` 与 `get` 均线程安全；
  但取出的 `ops+ctx` 在调用期间的线程安全由具体 BSP 保证。
- **BSP ops 的可重入性**须在头文件注释中标注：是否可并发调用、是否持锁。
  共享总线（I2C/SPI）的 BSP 应在 ctx 内部加总线锁。
- **ISR 与共享变量**：BSP 内部若 ISR 与任务共享状态，必须用 `volatile` + 临界区/原子操作；
  多字节访问须保证完整性。
- **硬实时路径**：严禁在 ops 的实时关键段内加日志/断点；时间测量用 GPIO 翻转 + 逻辑分析仪。
- **Watchdog**：所有 Service 任务应注册到 `esp_task_wdt`，避免死循环卡死系统。

## 7. 总结

本架构通过**设备管理**、**延迟初始化**与**接口注入**的有机结合，实现了：

1. **硬件与业务彻底解耦**：Service 层对底层芯片型号完全盲化，换芯片仅需改 BSP
2. **灵活的扩展能力**：新增外设仅需新增 BSP + DAL 接口，上层代码零侵入
3. **清晰的职责边界**：BSP 懂芯片驱动（仅绑定 ops + ctx），板级组装器懂设备注册，DAL 懂设备管理，Middleware 懂算法，SVC 懂业务逻辑与初始化时机
4. **延迟硬件初始化**：`create()` 零副作用，硬件上电由 Service 层按需触发 `ops->init()`，降低启动冲击与启动失败风险
5. **统一的错误语义**：`dal_err_t` 在 BSP 边界隔离底层码，上层只见业务错误
6. **简约务实**：不引入 OSAL/PAL 冗余抽象层，BSP 直接使用 FreeRTOS + ESP-IDF 原生 API

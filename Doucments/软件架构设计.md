# 嵌入式混合架构工程实施规范

## 1. 架构概述

本规范定义了一套基于"设备管理 + 接口注入"混合架构的嵌入式系统开发标准。通过严格的分层设计与契约化编程，实现硬件驱动与业务逻辑的彻底解耦，确保系统的可测试性、可扩展性与长期可维护性。

### 1.1 架构图

```
+---------------------+
|     Service         |   业务逻辑层（依赖注入，由 Assembler 组装）
+---------------------+
|     Middleware      |   中间件层（协议、算法、通用组件）
+---------------------+
|       DAL           |   设备抽象层（纯业务语义契约 + 设备注册表）
+---------------------+
|       BSP           |   板级适配层（硬件适配与翻译，自注册到 DAL）
+---------------------+
|     PAL / OSAL      |   平台屏蔽层 / OS 屏蔽层
+---------------------+
```

### 1.2 核心思想

- **设备管理**：BSP 向 DAL 注册物理设备，DAL 仅维护"业务功能"注册表，不感知具体芯片型号。
- **接口注入**：Service 声明能力需求（Deps），Assembler (main.c) 从 DAL 获取实现并完成依赖注入。
- **参数隔离**：所有跨层接口参数必须为纯业务语义，严禁泄露底层硬件类型。
- **错误码隔离**：底层 HAL/SDK 错误码（如 `esp_err_t`、负数 errno）必须在 BSP 边界翻译为统一的 `dal_err_t`，禁止透传到 DAL/SVC。

### 1.3 各层职责

| 层 | 职责 | 是否感知硬件 |
|----|------|--------------|
| **PAL** | 平台屏蔽层。封装 MCU SDK 的 GPIO/I2C/SPI/Timer 等基础外设 API，向上提供统一原语（如 `pal_gpio_write`）。 | 是（封装在内） |
| **OSAL** | OS 屏蔽层。封装 RTOS 接口（mutex/semaphore/queue/timer/task），向上提供 `osal_xxx` 统一接口，便于在 FreeRTOS/裸机/Host 间移植。 | 否 |
| **BSP** | 板级适配层。针对具体板卡，组合 PAL 原语驱动真实芯片，实现 DAL ops，并**自注册**到 DAL。仅 BSP 知道引脚号、I2C 地址、芯片型号。 | 是 |
| **DAL** | 设备抽象层。定义纯业务语义接口契约（`dal_xxx_interface.h`）+ 设备注册表管理（`dal_xxx.h/.c`）。不依赖任何 BSP。 | 否 |
| **Middleware** | 中间件层。与具体硬件无关的通用组件，如协议解析（Modbus/MQTT）、数据格式转换、滤波算法等。可被 Service 复用。 | 否 |
| **Service** | 业务逻辑层。通过依赖注入获得 DAL ops，编排业务流程。对底层芯片型号完全盲化。 | 否 |

## 3. 核心架构实现

### 3.1 DAL 设备管理实现

#### 3.1.1 公共错误码定义

> ⚠️ **关键约束**：`dal_err_t` 是 DAL/SVC 层唯一允许的错误码。BSP 必须在 ops 边界把底层码（`esp_err_t`、PAL 返回码等）翻译为 `dal_err_t`，原始码不得透传。

```c
/**
 * @file    dal_err.h
 * @brief   DAL 公共错误码定义
 *
 * @author  xiamu
 * @version 1.0
 */
#ifndef DAL_ERR_H
#define DAL_ERR_H

#ifdef __cplusplus
extern "C" {
#endif

/** DAL 统一错误码 */
typedef enum {
    DAL_OK              =  0,  /**< 成功 */
    DAL_ERR_INVALID     = -1,  /**< 参数非法 */
    DAL_ERR_NO_MEM      = -2,  /**< 无内存 / 注册表满 */
    DAL_ERR_NOT_FOUND   = -3,  /**< 设备未注册 */
    DAL_ERR_BUSY        = -4,  /**< 设备忙 */
    DAL_ERR_TIMEOUT     = -5,  /**< 超时 */
    DAL_ERR_HW          = -6,  /**< 底层硬件错误（已翻译，不泄露原始码） */
    DAL_ERR_STATE       = -7,  /**< 状态非法（如重复注册） */
    DAL_ERR_UNSUPPORTED = -8,  /**< 操作不支持 */
} dal_err_t;

#ifdef __cplusplus
}
#endif
#endif /* DAL_ERR_H */
```

#### 3.1.2 接口契约定义（纯业务语义）

> ⚠️ **关键约束**：契约中严禁出现 `i2c_port_t`、`gpio_num_t`、`GPIO_TypeDef` 等硬件类型。所有硬件上下文必须封装在不透明指针 `void *ctx` 中。

```c
/**
 * @file    dal_display_interface.h
 * @brief   显示设备 DAL 接口契约（纯业务语义）
 *
 * @note    本文件为 Service 层与 BSP 层的共同契约。
 *          - Service 仅包含本文件，不得包含 dal_display.h（管理 API）。
 *          - 本文件严禁出现任何硬件 SDK 类型。
 *
 * @author  xiamu
 * @version 1.0
 */
#ifndef DAL_DISPLAY_INTERFACE_H
#define DAL_DISPLAY_INTERFACE_H

#include "dal_err.h"
#include <stdint.h>
#include <stddef.h>

#ifdef __cplusplus
extern "C" {
#endif

/** 显示设备配置（纯业务语义） */
typedef struct {
    uint16_t width;        /**< 像素宽 */
    uint16_t height;       /**< 像素高 */
    uint8_t  brightness;   /**< 背光百分比 0~100 */
} dal_display_config_t;

/** 显示设备操作契约 */
typedef struct {
    dal_err_t (*init)(void *ctx, const dal_display_config_t *cfg);             /**< 初始化 */
    dal_err_t (*flush_frame)(void *ctx, const uint8_t *rgb_buf, size_t len);   /**< 刷新一帧 */
    dal_err_t (*set_backlight)(void *ctx, uint8_t percent);                    /**< 设置背光 */
} dal_display_ops_t;

#ifdef __cplusplus
}
#endif
#endif /* DAL_DISPLAY_INTERFACE_H */
```

#### 3.1.4 设备管理实现

```c
/**
 * @file    dal_display.c
 * @brief   显示设备 DAL 管理实现（注册表 + 互斥保护）
 *
 * @author  xiamu
 * @version 1.0
 */
#include "dal_display.h"
#include "osal_mutex.h"
#include <string.h>

#define MAX_DEVICES 4   /**< 可由 Kconfig 覆盖 */

typedef struct {
    const char              *name;
    const dal_display_ops_t *ops;
    void                    *ctx;
} device_entry_t;

static device_entry_t s_registry[MAX_DEVICES];
static size_t         s_count = 0;
static osal_mutex_t   s_mutex;
static bool           s_initialized = false;

/* 注册表互斥锁在模块加载期一次性创建，避免懒初始化的竞态 */
__attribute__((constructor))
static void registry_once_init(void)
{
    osal_mutex_create(&s_mutex);   /* 宿主侧无 constructor 时由 dal_display_init() 显式调用 */
}

dal_err_t dal_display_register(const char *name,
                               const dal_display_ops_t *ops,
                               void *ctx)
{
    if (!name || !ops) {
        return DAL_ERR_INVALID;
    }

    osal_mutex_lock(s_mutex, OSAL_WAIT_FOREVER);
    if (s_count >= MAX_DEVICES) {
        osal_mutex_unlock(s_mutex);
        return DAL_ERR_NO_MEM;
    }
    for (size_t i = 0; i < s_count; i++) {
        if (strcmp(s_registry[i].name, name) == 0) {
            osal_mutex_unlock(s_mutex);
            return DAL_ERR_STATE;   /* 名称冲突 */
        }
    }

    s_registry[s_count].name = name;
    s_registry[s_count].ops  = ops;
    s_registry[s_count].ctx  = ctx;
    s_count++;
    osal_mutex_unlock(s_mutex);
    return DAL_OK;
}

dal_err_t dal_display_get(const char *name,
                          const dal_display_ops_t **ops,
                          void **ctx)
{
    if (!name || !ops || !ctx) {
        return DAL_ERR_INVALID;
    }

    *ops = NULL;
    *ctx = NULL;

    osal_mutex_lock(s_mutex, OSAL_WAIT_FOREVER);
    for (size_t i = 0; i < s_count; i++) {
        if (strcmp(s_registry[i].name, name) == 0) {
            *ops = s_registry[i].ops;
            *ctx = s_registry[i].ctx;
            osal_mutex_unlock(s_mutex);
            return DAL_OK;
        }
    }
    osal_mutex_unlock(s_mutex);
    return DAL_ERR_NOT_FOUND;
}
```

> **设计要点**：
> - 互斥锁在模块初始化期一次性创建，杜绝原懒初始化方案在并发 `register` 时的竞态。
> - `dal_display_get` 合并 ops 与 ctx 一次取出，避免分两次调用期间设备被注销导致的不一致。

### 3.2 BSP 单驱动注册

> **规范范式**：BSP 在 `bsp_xxx_init()` 内部完成硬件初始化后，**自注册**到 DAL。main 仅依次调用各 `bsp_xxx_init()`，不参与注册动作。

```c
/**
 * @file    bsp_relay.c
 * @brief   Board V1 继电器 BSP — GPIO 后端实现 + 自注册到 DAL
 *
 * @author  xiamu
 * @version 1.0
 */
#include "bsp_relay.h"
#include "dal_relay.h"          /* DAL 管理 API（含 register） */
#include "dal_relay_interface.h"/* relay 接口契约 */
#include "pal_gpio.h"
#include <stdbool.h>
#include <stddef.h>

/* ====== GPIO 继电器硬件上下文（BSP 私有）====== */
typedef struct {
    int  gpio_pin;
    bool active_high;
} bsp_relay_hw_ctx_t;

/* ====== PAL 返回码 → dal_err_t 翻译（BSP 边界）====== */
static dal_err_t relay_err_from_pal(int pal_ret)
{
    if (pal_ret == 0) return DAL_OK;
    if (pal_ret == -1) return DAL_ERR_INVALID;
    return DAL_ERR_HW;
}

/* ====== GPIO 继电器 ops 实现（BSP 私有）====== */
static dal_err_t bsp_relay_gpio_set(void *ctx, bool on)
{
    const bsp_relay_hw_ctx_t *hw = (const bsp_relay_hw_ctx_t *)ctx;
    int level = on ? (hw->active_high ? 1 : 0)
                   : (hw->active_high ? 0 : 1);
    return relay_err_from_pal(pal_gpio_write(hw->gpio_pin, level));
}

static dal_err_t bsp_relay_gpio_get(void *ctx, bool *out)
{
    const bsp_relay_hw_ctx_t *hw = (const bsp_relay_hw_ctx_t *)ctx;
    if (!out) return DAL_ERR_INVALID;
    int level = pal_gpio_read(hw->gpio_pin);
    if (level < 0) return relay_err_from_pal(level);

    *out = (hw->active_high ? (level == 1) : (level == 0));
    return DAL_OK;
}

/* BSP 私有的 GPIO ops，不对外暴露 */
static const dal_relay_ops_t s_gpio_relay_ops = {
    .set = bsp_relay_gpio_set,
    .get = bsp_relay_gpio_get,
};

/* ====== 本板硬件实例（仅此处知道引脚号）====== */
static bsp_relay_hw_ctx_t s_lock_ctx    = { .gpio_pin = 1, .active_high = false };
static bsp_relay_hw_ctx_t s_alarm_ctx   = { .gpio_pin = 2, .active_high = false };
static bsp_relay_hw_ctx_t s_wiegand_ctx = { .gpio_pin = 3, .active_high = true  };

/* ====== BSP 对外初始化接口（自注册）====== */
dal_err_t bsp_relay_init(void)
{
    /* 1. GPIO 硬件初始化 + 置安全默认态 */
    const bsp_relay_hw_ctx_t *all_ctx[] = { &s_lock_ctx, &s_alarm_ctx, &s_wiegand_ctx };
    for (size_t i = 0; i < sizeof(all_ctx) / sizeof(all_ctx[0]); i++) {
        pal_gpio_set_direction(all_ctx[i]->gpio_pin, PAL_GPIO_DIR_OUTPUT);
        int off_level = all_ctx[i]->active_high ? 0 : 1;
        pal_gpio_write(all_ctx[i]->gpio_pin, off_level);
    }

    /* 2. 以业务语义名称自注册到 DAL */
    dal_err_t ret;
    ret = dal_relay_register("door_lock",   &s_gpio_relay_ops, &s_lock_ctx);
    if (ret != DAL_OK) return ret;

    ret = dal_relay_register("alarm",       &s_gpio_relay_ops, &s_alarm_ctx);
    if (ret != DAL_OK) return ret;

    ret = dal_relay_register("wiegand_pwr", &s_gpio_relay_ops, &s_wiegand_ctx);
    if (ret != DAL_OK) return ret;

    return DAL_OK;
}
```

```c
/**
 * @file    bsp_relay.h
 * @brief   Board V1 继电器板级支持包 - 公共接口
 *
 * @note    此文件仅声明本板继电器子系统的初始化入口。
 *          - 不包含任何 GPIO 引脚定义
 *          - 不包含 dal_relay_ops_t 或硬件上下文结构体
 *          - Service 层不应包含此文件（Service 应直接使用 dal_relay_interface.h）
 *
 * @author  xiamu
 * @version 1.0
 */
#ifndef BSP_RELAY_H
#define BSP_RELAY_H

#include "dal_err.h"

#ifdef __cplusplus
extern "C" {
#endif

/**
 * @brief 初始化本板所有继电器硬件并自注册到 DAL
 *
 * @details 完成以下工作：
 *          1. 配置所有继电器 GPIO 为输出模式并置于安全默认状态
 *          2. 将每个继电器以语义化名称注册到 DAL relay 模块

 * @retval DAL_ERR_HW      GPIO 初始化失败
 * @retval DAL_ERR_STATE   DAL 注册名称冲突

 */
dal_err_t bsp_relay_init(void);

#ifdef __cplusplus
}
#endif
#endif /* BSP_RELAY_H */
```

### 3.3 BSP 多芯片驱动注册

#### 3.3.2 主 BSP 组装器

```c
/**
 * @file    bsp_display_rpi7pin.c
 * @brief   RPI 7 寸屏 BSP 组装器 — 聚合 TC358762 + ATTINY88，自注册到 DAL
 * @author  xiamu
 * @version 1.0
 */
#include "bsp_display_rpi7pin.h"
#include "dal_display.h"               /* DAL 管理 API */
#include "bsp_display_attiny88.h"      /* 子芯片内部头 */
#include "bsp_display_tc358762.h"      /* 子芯片内部头 */
#include <stdlib.h>

/* ====== esp_err_t → dal_err_t 翻译（BSP 边界）====== */
static dal_err_t display_err_from_esp(esp_err_t e)
{
    switch (e) {
        case ESP_OK:               return DAL_OK;
        case ESP_ERR_NO_MEM:       return DAL_ERR_NO_MEM;
        case ESP_ERR_TIMEOUT:      return DAL_ERR_TIMEOUT;
        case ESP_ERR_INVALID_ARG:  return DAL_ERR_INVALID;
        default:                   return DAL_ERR_HW;
    }
}

/* ====== 聚合上下文：封装所有子芯片句柄 ====== */
typedef struct {
    bsp_tc358762_ctx_t *bridge;
    bsp_attiny88_ctx_t *backlight_mcu;
    uint16_t            width;
    uint16_t            height;
} bsp_rpi7pin_agg_ctx_t;

/* ====== 适配层：将 DAL 接口调用分发到对应子芯片 ====== */
static dal_err_t rpi7pin_init(void *ctx, const dal_display_config_t *cfg)
{
    bsp_rpi7pin_agg_ctx_t *agg = (bsp_rpi7pin_agg_ctx_t *)ctx;
    dal_err_t ret;

    ret = bsp_tc358762_init(&agg->bridge, cfg->width, cfg->height);
    if (ret != DAL_OK) return ret;

    ret = bsp_attiny88_init(&agg->backlight_mcu, 0x3A);
    if (ret != DAL_OK) {
        bsp_tc358762_deinit(agg->bridge);
        agg->bridge = NULL;
        return ret;
    }

    agg->width  = cfg->width;
    agg->height = cfg->height;
    return bsp_attiny88_set_backlight(agg->backlight_mcu, cfg->brightness);
}

static dal_err_t rpi7pin_flush(void *ctx, const uint8_t *rgb_buf, size_t len)
{
    bsp_rpi7pin_agg_ctx_t *agg = (bsp_rpi7pin_agg_ctx_t *)ctx;
    return bsp_tc358762_flush(agg->bridge, rgb_buf, len);
}

static dal_err_t rpi7pin_set_bl(void *ctx, uint8_t percent)
{
    bsp_rpi7pin_agg_ctx_t *agg = (bsp_rpi7pin_agg_ctx_t *)ctx;
    return bsp_attiny88_set_backlight(agg->backlight_mcu, percent);
}

static const dal_display_ops_t s_rpi7pin_ops = {
    .init          = rpi7pin_init,
    .flush_frame   = rpi7pin_flush,
    .set_backlight = rpi7pin_set_bl,
};

/* ====== 对外初始化入口（自注册）====== */
dal_err_t bsp_display_rpi7pin_init(void)
{
    bsp_rpi7pin_agg_ctx_t *agg = calloc(1, sizeof(bsp_rpi7pin_agg_ctx_t));
    if (!agg) return DAL_ERR_NO_MEM;

    /* 预初始化使设备进入可用态；运行参数后续可由 Service 通过 ops.init 重配 */
    dal_display_config_t default_cfg = {
        .width = 800, .height = 480, .brightness = 80
    };

    dal_err_t ret = rpi7pin_init(agg, &default_cfg);
    if (ret != DAL_OK) {
        free(agg);
        return ret;
    }

    ret = dal_display_register("rpi7pin", &s_rpi7pin_ops, agg);
    if (ret != DAL_OK) {
        bsp_tc358762_deinit(agg->bridge);
        bsp_attiny88_deinit(agg->backlight_mcu);
        free(agg);
        return ret;
    }
    return DAL_OK;
}
```

#### 3.4.2 业务逻辑实现

```c
/**
 * @file    svc_ui.c
 * @brief   UI Service 业务逻辑实现
 * @author  xiamu
 * @version 1.0
 */
#include "svc_ui.h"
#include "osal_mutex.h"
#include <string.h>

static svc_ui_deps_t s_deps;        /* 值拷贝 deps 结构体本身 */
static osal_mutex_t  s_mutex;
static bool          s_inited = false;

dal_err_t svc_ui_init(const svc_ui_deps_t *deps)
{
    if (!deps || !deps->display_ops || !deps->display_ops->flush_frame) {
        return DAL_ERR_INVALID;
    }

    /* 值拷贝 deps 结构体本身（注意：结构体内是指针，拷贝的是指针值，
       依赖调用方保证 ops/ctx 生命周期覆盖 Service 运行期） */
    memcpy(&s_deps, deps, sizeof(s_deps));

    if (osal_mutex_create(&s_mutex) != OSAL_OK) {
        return DAL_ERR_HW;
    }
    s_inited = true;
    return DAL_OK;
}
```

### 3.5 Assembler 组装实现

> **职责**：main.c 仅负责**按序调用各 BSP init**（触发自注册）、**从 DAL 取出接口**、**注入 Service**。不直接接触任何硬件、不直接调用 ops。

```c
/**
 * @file    main.c
 * @brief   Assembler：启动顺序、注册触发、依赖注入
 * @author  xiamu
 * @version 1.0
 */
#include "dal_display.h"            /* DAL 管理 API（get） */
#include "dal_relay.h"              /* DAL 管理 API（get） */
#include "bsp_relay.h"              /* BSP init 入口 */
#include "bsp_display_rpi7pin.h"    /* BSP init 入口 */
#include "svc_ui.h"                 /* Service 依赖契约 */

void app_main(void)
{
    /* ====== Step 1: BSP 自注册到 DAL（内部完成硬件初始化 + dal_register）====== */
    if (bsp_relay_init() != DAL_OK) {
        /* 错误处理... */
        return;
    }
    if (bsp_display_rpi7pin_init() != DAL_OK) {
        /* 错误处理... */
        return;
    }

    /* ====== Step 2: 从 DAL 取出接口，注入给 Service ====== */
    const dal_display_ops_t *disp_ops = NULL;
    void                    *disp_ctx = NULL;
    if (dal_display_get("rpi7pin", &disp_ops, &disp_ctx) != DAL_OK) {
        /* 错误处理... */
        return;
    }

    svc_ui_deps_t deps = {
        .display_ops = disp_ops,
        .display_ctx = disp_ctx,
    };
    if (svc_ui_init(&deps) != DAL_OK) {
        /* 错误处理... */
        return;
    }

    /* ====== Step 3: 启动业务（创建任务 / 触发主循环）====== */
}
```

## 4. 架构规范约束

### 4.1 依赖方向约束

**编译期依赖（头文件包含）**：

```
main.c     → BSP(管理头) + DAL(管理头) + Service(契约头)
Service    → DAL(接口契约头) + OSAL + Middleware
BSP        → DAL(管理头) + PAL + OSAL
DAL        → OSAL + dal/common   （DAL 管理实现仅依赖 OSAL，严禁依赖任何 BSP）
Middleware → OSAL + dal/common
```

**注册期 / 运行期调用**：

```
main → bsp_xxx_init()     （BSP 内部自注册到 DAL）
main → dal_xxx_get()      （取 ops+ctx 注入 Service）
main → svc_xxx_init(&deps)
```

**禁止的依赖**：

- Service → DAL 管理头（`dal_xxx.h`）—— 只允许 `dal_xxx_interface.h`
- Service → BSP / PAL / 硬件 SDK 头文件
- DAL → BSP —— DAL 不得反向依赖任何具体 BSP，否则失去设备无关性

> ⚠️ 注意：Service 依赖 DAL 的**接口契约**是允许且必须的；被禁止的是依赖 DAL 的**管理实现**与 BSP/PAL。两者不可混淆。

### 4.2 头文件包含规范

#### 4.2.1 Service 层禁止包含

```c
// ❌ 错误示例
#include "dal_display.h"     // 禁止依赖 DAL 管理头
#include "bsp_xxx.h"         // 禁止依赖 BSP
#include "pal_xxx.h"         // 禁止依赖 PAL
#include "driver/i2c.h"      // 禁止依赖硬件 SDK 头文件
```

#### 4.2.2 Service 层允许包含

```c
// ✅ 正确示例
#include "svc_xxx.h"                // 自身依赖契约
#include "dal_xxx_interface.h"      // 仅允许依赖 DAL 接口契约
#include "osal_xxx.h"               // OS 抽象层
#include "mw_xxx.h"                 // 中间件
```

### 4.3 构建系统约束

```cmake
# service/CMakeLists.txt
idf_component_register(
    SRCS "svc_ui.c"
    INCLUDE_DIRS "include"
    PRIV_REQUIRES
        osal
        dal_display          # 仅链接 DAL 接口契约组件（interface.h）
    # ❌ 禁止声明 bsp_xxx / pal_xxx 或任何硬件驱动组件
)
```

### 4.4 生命周期与初始化顺序

系统启动必须遵循以下严格顺序，违反顺序会导致注册表为空或依赖悬空：

```
1. OSAL / RTOS 启动（内核调度已就绪）
2. PAL 初始化（基础外设原语可用）
3. 各 BSP init() 依次调用          → 触发自注册，DAL 注册表填充完成
4. 各 Service init(&deps) 调用      → Assembler 先 dal_get 再注入
5. 启动业务任务 / 主循环
```

- **注册表必须在任何 Service init 之前完成填充**，否则 `dal_get` 返回 `DAL_ERR_NOT_FOUND`。
- BSP init 之间若存在硬件依赖（如共享 I2C 总线），按依赖顺序排列，并在 BSP 内部自行确保总线先就绪。
- 析构顺序与初始化相反：先停业务任务 → Service deinit → BSP deinit（注销）→ PAL deinit。

## 5. 接口设计防泄漏检查清单

为防止硬件细节通过参数或返回值泄露到上层，所有 DAL/SVC 接口设计必须通过以下检查：

- [ ] 接口参数中无硬件枚举（如 `i2c_port_t`、`gpio_num_t`）
- [ ] 接口参数中无硬件寄存器结构体指针
- [ ] 复杂硬件配置已替换为纯业务语义结构体或不透明指针 `void *`
- [ ] 返回值统一为 `dal_err_t`，无底层 HAL/SDK 错误码（`esp_err_t` 等）透传
- [ ] BSP 在 ops 边界完成错误码翻译
- [ ] Service 层编译不依赖任何 BSP/PAL 头文件
- [ ] DAL 接口契约头不 `#include` 任何硬件 SDK 头文件

## 7. 并发与中断约束

- **DAL ops 默认运行在任务上下文**，禁止在 ISR 中直接调用 ops（多数 ops 含阻塞/互斥）。如需 ISR 触发，应在 ISR 中仅投递消息到队列，由任务上下文消费并调用 ops。
- **DAL 注册表内部用 OSAL 互斥锁保护**，`register` 与 `get` 均线程安全；但取出的 `ops+ctx` 在调用期间的线程安全由具体 BSP 保证（见下）。
- **BSP ops 的可重入性**须在头文件 `@note` 中标注：是否可并发调用、是否持锁。共享总线（I2C/SPI）的 BSP 应在 ctx 内部加总线锁。
- **ISR 与共享变量**：BSP 内部若 ISR 与任务共享状态，必须用 `volatile` + 临界区/原子操作；多字节访问须保证完整性。
- **硬实时路径**：严禁在 ops 的实时关键段内加日志/断点；时间测量用 GPIO 翻转 + 逻辑分析仪。

## 8. 演进与维护

### 8.1 新增板卡/芯片

1. 在 `bsp/` 下新建适配文件，组合 PAL 原语驱动新芯片
2. 复用现有 DAL 接口契约（无需修改 DAL）
3. 在新 BSP 的 `init()` 中根据 Kconfig 注册对应实例
4. `main.c` 增加一行 `bsp_xxx_init()` 调用；Service 层代码零修改

### 8.2 接口变更与版本兼容

1. 扩展 `deps_t` / `ops_t` 字段时，**仅在结构体末尾追加**，不得调整已有字段顺序
2. 在 ops/配置结构体首字段增加 `uint32_t version`，用于运行时兼容校验：

```c
typedef struct {
    uint32_t version;            /**< ABI 版本，由 DAL 定义当前宏 */
    uint16_t width;
    uint16_t height;
    uint8_t  brightness;
} dal_display_config_t;

#define DAL_DISPLAY_CFG_VERSION  2

/* Service 侧校验 */
if (cfg->version < DAL_DISPLAY_CFG_VERSION) {
    return DAL_ERR_UNSUPPORTED;
}
```

3. 更新 `main.c` 适配代码，保持向后兼容（旧字段语义不变）
4. 任何破坏性变更必须升级 `version` 宏并在迁移文档中记录

## 9. 总结

本混合架构通过**设备管理**与**接口注入**的有机结合，实现了：

1. **硬件与业务彻底解耦**：Service 层对底层芯片型号完全盲化
2. **极致的可测试性**：纯业务语义契约使 Host 侧 Mock 测试顺理成章
3. **灵活的扩展能力**：新增芯片仅需新增/修改 BSP，上层代码零侵入
4. **清晰的职责边界**：BSP 懂芯片，DAL 懂设备管理，SVC 懂业务逻辑
5. **统一的错误语义**：`dal_err_t` 在 BSP 边界隔离底层码，上层只见业务错误
